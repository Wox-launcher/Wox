# 应用插件

应用搜索是全局插件。直接输入应用名即可，不需要关键字。

## 快速开始

```text
chrome
visual studio code
settings
```

按 `Enter` 打开选中的应用。如果 Wox 能识别该应用已经运行，结果可以激活现有窗口，而不是再启动一个实例。

![应用插件结果列表](/images/system-plugin-app.png)

## 动作

在应用结果上打开 [操作面板](../../usage/action-panel.md)，可以执行二级动作：

| 动作 | 用途 |
| --- | --- |
| 打开 | 启动或激活应用 |
| 打开所在文件夹 | 在系统文件管理器中显示应用文件 |
| 复制路径 | 复制可执行文件或 app bundle 路径 |
| 显示上下文菜单 | 在支持的平台上打开系统上下文菜单 |
| 终止应用 | 在 Wox 能识别进程时停止运行中的应用 |

## 应用来源

Wox 会按平台索引常见应用位置：

| 平台 | 来源 |
| --- | --- |
| Windows | 开始菜单、`Program Files`、WindowsApps、UWP 应用、Windows 设置页 |
| macOS | `/Applications`、`.app` bundle、系统设置面板 |
| Linux | `.desktop` 文件和配置的应用目录 |

## 设置

打开 **设置 -> 插件 -> 应用**，可以添加自定义应用目录或手动重新索引。便携应用或本地工具不在标准应用目录里时，这个设置最有用。

## 排序

应用插件会结合匹配分数和 MRU 数据排序。经常启动的应用会逐渐靠前，所以正常使用一段时间后，第一条结果会更稳定。

## 排查

### 新安装的应用搜不到

- 等待几秒，让索引完成。
- 运行插件的重新索引命令，或重启 Wox。
- 如果是便携应用，把应用所在目录加到插件设置里。

### 排在第一的不是我要的应用

正常打开正确结果几次。MRU 会把经常使用的应用排到相似匹配前面。

### CPU 或内存信息不显示

只有当 Wox 能把结果映射到运行中的桌面进程时，才会显示运行时信息。商店应用、设置面板或某些桌面环境可能不提供这些信息。
