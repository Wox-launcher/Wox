# Application 插件

Application 插件让你可以快速搜索和启动系统中的应用程序。

## 功能特点

- **快速启动**: 直接输入应用名称即可启动
- **智能匹配**: 支持模糊匹配应用名称和路径
- **最近使用优先**: MRU 功能会记住你常用的应用
- **运行状态显示**: 显示正在运行的应用及其 CPU 和内存占用
- **多平台支持**: 支持 Windows、macOS 和 Linux

## 基本使用

### 搜索和启动应用

1. 打开 Wox（默认快捷键 `Alt+Space` 或 `Cmd+Space`）
2. 直接输入应用名称，如：
   - `Chrome` - 启动 Google Chrome
   - `VSCode` - 启动 Visual Studio Code
   - `Notion` - 启动 Notion

3. 按下 `Enter` 键启动选中的应用

### 操作正在运行的应用

如果应用正在运行，搜索结果会显示 CPU 和内存占用信息：

```
Chrome
CPU: 2.5%
Memory: 450.3 MB
```

### 应用操作

选中应用后，可以执行以下操作：

| 操作 | 快捷键 | 说明 |
|-----|--------|------|
| 打开应用 | `Enter` | 启动应用（如果已运行则激活窗口） |
| 打开所在文件夹 | - | 在文件管理器中打开应用所在目录 |
| 复制路径 | - | 复制应用完整路径到剪贴板 |
| 显示右键菜单 | `Ctrl+M` | 显示系统右键菜单（仅桌面应用） |
| 终止应用 | - | 终止正在运行的应用 |

## 高级功能

### 自定义应用目录

默认情况下，Application 插件会自动索引系统中的应用程序目录。你可以添加自定义目录来包含更多应用：

1. 打开 Wox 设置
2. 找到 **Application** 插件
3. 点击进入插件设置
4. 在 **应用目录** 表格中添加自定义路径
5. 点击保存

**注意**:
- 支持递归搜索子目录
- 递归深度默认为 3 层
- 可以设置排除特定目录

### 应用类型

Application 插件支持多种应用类型：

- **桌面应用**: `.app`（macOS）、`.exe`（Windows）、Linux 可执行文件
- **UWP 应用**: Windows Store 应用
- **系统设置**: Windows 设置页面（仅 Windows）
- **macOS 系统设置**: 系统偏好设置面板（仅 macOS）

### MRU 功能

Application 插件支持 MRU（最近使用）功能：

- 经常使用的应用会优先显示
- 基于 MRU 数据进行智能排序
- 可以在设置中查看 MRU 使用统计

## 平台特性

### Windows

- 自动索引常用应用目录：
  - `C:\Program Files`
  - `C:\Program Files (x86)`
  - `%APPDATA%\Microsoft\Windows\Start Menu`
  - `%LOCALAPPDATA%\Microsoft\WindowsApps`
- 支持 UWP 应用（Microsoft Store 应用）
- 支持系统设置页面
- 如果应用已运行，会尝试激活其窗口

### macOS

- 自动索引 `/Applications` 目录
- 支持 `.app` 应用包
- 支持系统偏好设置面板
- 默认行为：如果应用已运行，会激活其窗口

### Linux

- 支持系统默认应用目录
- 支持 `.desktop` 文件
- 可能需要手动配置应用目录

## 常见问题

### 为什么搜索不到某个应用？

1. **检查应用是否在索引目录中**：在插件设置中查看应用目录列表
2. **等待索引完成**：新安装的应用可能需要几秒钟才能被索引
3. **重启 Wox**：有时需要重启才能重新索引应用

### 如何隐藏不需要的应用？

Application 插件本身不提供隐藏功能，但可以通过以下方式：
- 不将应用放在索引目录中
- 使用第三方插件管理可见应用

### 应用启动很慢怎么办？

1. 检查应用的启动路径是否正确
2. 确认应用文件没有被移动或删除
3. 查看系统资源占用，确保有足够资源启动应用

### 为什么 CPU 和内存信息不显示？

- 确保应用正在运行
- 仅桌面应用会显示 CPU 和内存信息
- Windows UWP 应用和 macOS 系统设置不显示这些信息
