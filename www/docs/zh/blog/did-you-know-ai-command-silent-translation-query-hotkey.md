# 你知道吗：Wox 可以用快捷键静默翻译选中文本

AI 命令本身可以在启动器里使用，但如果和快捷键查询组合起来，就会更接近日常文本处理工具：在任意应用里选中文本，按一个快捷键，让 AI 翻译，最后由 Wox 把最终结果直接粘贴回原来的选区。

关键点是给 AI 命令配置明确的默认动作。在 AI 命令插件设置里，新增或编辑一个翻译命令：

| AI 命令字段 | 示例值 |
| --- | --- |
| 名称 | `翻译为中文` |
| 命令 | `translate` |
| 提示词 | `Translate the following text to Chinese. Return only the translated text: %s` |
| 默认动作 | `运行并粘贴` |

`运行并粘贴` 会等待 AI 返回最终结果，然后把最终文本写入剪贴板，并模拟粘贴回 Wox 打开前记录的活动窗口。它不会粘贴 streaming 过程中的半成品。

然后，新增一个快捷键查询，选择 **静默执行** 预设，再把它绑定到这条命令：

| 快捷键查询字段 | 示例值 |
| --- | --- |
| 预设 | `静默执行` |
| 快捷键 | 任意可用快捷键，例如 `ctrl+shift+t` |
| 查询 | `ai translate {wox:selected_text}` |
| 可选调整 | 只有在你需要覆盖显示行为时，再切到 `自定义` 继续修改 |

静默执行这个预设会默认开启静默执行，所以大多数情况下只需要填写快捷键和查询。它本身不包含隐藏的粘贴语义；在这个配置里，静默执行只是运行 AI 命令的默认动作，而这个默认动作被你显式设置成了 `运行并粘贴`。

完整流程会变成：

1. 在其他应用里选中文本。
2. 按下快捷键查询。
3. Wox 把选中文本发送给 AI 命令。
4. Wox 在鼠标附近显示一个很轻的思考提示。
5. AI 返回最终答案后，Wox 用翻译结果替换原来的选中文本。

同样的模式也适合其他文本改写：语法修正、语气调整、摘要、格式清理等。如果你想先检查结果，就把默认动作设为 `运行`；只有当这个命令足够适合直接替换选中文本时，再使用 `运行并粘贴`。
